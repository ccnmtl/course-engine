What is Course Engine?

Course Engine lets you create and manage EdX courses using Excel spreadsheets ‚Äî no coding or XML editing required. It supports two workflows:

üî®

Build Mode

Create a new EdX course from an Excel workbook. Upload your .xlsx file, preview the course structure, and export a .tar.gz archive ready to import into EdX Studio.

üì•

Import Mode

Convert an existing EdX course export back into Excel format. Upload a .tar.gz from EdX Studio, and download a fully formatted .xlsx workbook for editing.

üîí Privacy: All processing happens entirely in your browser. No data is ever uploaded to any server. Your course content stays on your computer.

Building a Course from Excel

  1. Prepare your Excel workbook

    Use the Download Template button on the home screen to get a pre-filled workbook. The workbook must contain 6 sheets: Course Info, Structure, Text Blocks, Videos, Problems, and Open Response.

  2. Upload the workbook

    Drag and drop your .xlsx file onto the upload zone, or click Browse Files. The parser will validate all sheets and flag any errors or missing references.

  3. Preview your course

    Review the full course structure tree. It shows all chapters, subsections, units, and content blocks. Stats at the top show total counts of each type.

  4. Export for EdX

    Click Export .tar.gz for EdX to download the OLX archive. In EdX Studio, go to Tools ‚Üí Import and upload this file to create your course.

Building Course Structure

The Structure sheet is the backbone of your course. It defines the hierarchy: Sections ‚Üí Subsections ‚Üí Units ‚Üí Content blocks. Here's how to build a course structure step by step.

üí° EdX terminology: In EdX, a Section is called a chapter, a Subsection is called a sequential, and a Unit is called a vertical. The Structure sheet uses these EdX column names.

Step 1: Add a New Section

A section (chapter) is the top-level grouping in your course. To create one, add rows to the Structure sheet with the same value in the chapter column.

For example, to create a section called "Module 3: Assessment", every row that belongs to this section should have Module 3: Assessment in the chapter column.

Step 2: Add Subsections to the Section

Within a section, subsections (sequentials) group related units together. Fill in the sequential column to create subsections under your section.

For example, under "Module 3: Assessment" you might have two subsections: 3.1 Knowledge Check and 3.2 Final Quiz.

Step 3: Add Units and Content Blocks

Each row in the Structure sheet represents one content block inside a unit. The vertical column is the unit name, block_type tells the system what kind of content it is, and block_id links to the matching row in the content sheet (e.g., Problems, Text Blocks).

Step 4: Choose a Block ID

The block_id is a label you create yourself ‚Äî you can use any value you want. It's simply a unique identifier that links a row in the Structure sheet to its matching content row in another sheet (e.g., Problems or Text Blocks).

üìå Block ID rules:
‚Ä¢ You make it up ‚Äî there's no auto-generated ID. Just type a short, descriptive name.
‚Ä¢ Must be unique ‚Äî no two blocks in the entire workbook can share the same block_id.
‚Ä¢ Must match ‚Äî the block_id in the Structure sheet must exactly match the block_id in the corresponding content sheet.
‚Ä¢ Use letters, numbers, and underscores ‚Äî avoid spaces and special characters.
‚Ä¢ Be descriptive ‚Äî use a naming pattern that helps you identify the block later.

‚úÖ Good block_id examples

block_id Why it's good

mod2_quiz_q1 Module 2, quiz, question 1 ‚Äî includes context and numbering
intro_video Simple and self-explanatory for a single intro video
final_essay Clearly identifies the final essay open-response block

Step 5: Define the Content

For each block_id referenced in the Structure sheet, you must have a corresponding row in the appropriate content sheet. For problems, add a row to the Problems sheet with the same block_id. For text, add a row to Text Blocks, and so on.

Importing from EdX Studio

  1. Export from EdX Studio

    In EdX Studio, go to Tools ‚Üí Export and download the .tar.gz course archive to your computer.

  2. Upload the archive

    Switch to the Import from EdX tab on the home screen and drag your .tar.gz file onto the upload zone.

  3. Review the import

    Course Engine will parse all OLX XML files and display a preview of the course structure. Check the stats and structure to make sure everything looks correct.

  4. Download as Excel

    Click Export to Excel (.xlsx) to download the course as a formatted spreadsheet. You can edit it and re-export back to EdX using Build Mode.

Note: Some special EdX block types ‚Äî such as library content, discussion forums, and LTI integrations ‚Äî are skipped during import because they don't contain editable content in the export archive.

Tips & Best Practices

  • Start with the template: Download the template from the home screen ‚Äî it contains sample data showing the correct format for every sheet and block type.
  • Block IDs must be unique: Each block_id should be unique across the entire workbook. Use descriptive names like ch1_welcome_text or quiz_q1.
  • HTML content: Text block content supports full HTML including headings, paragraphs, lists, images, tables, and inline styles.
  • Round-trip editing: Export from EdX ‚Üí import to Excel ‚Üí edit the spreadsheet ‚Üí re-export back to EdX. This is a powerful way to bulk-edit course content.
  • Validation feedback: The parser checks for missing block IDs, duplicate IDs, and broken references between the Structure sheet and content sheets. Fix all errors before exporting.
  • YouTube video IDs: Use just the YouTube video ID (e.g. dQw4w9WgXcQ), not the full URL. The ID is the part after v= in the YouTube URL.
  • Problem answer format: Set the correct column to the letter of the correct choice (A, B, C, or D). Only single-answer problems are supported.
  • Self-paced courses: Setting Self-Paced to "Yes" in Course Info enables learners to progress at their own pace without fixed deadlines.
